🧩 Desafio de Indentação e Organização

Objetivo: Aprender a identificar se um código HTML está bem organizado e corretamente indentado, praticando a estruturação do código no seu próprio repositório GitHub.

🔍 Parte 1 — Observações da Inspeção

1. Acesse a página com código bagunçado publicada em:
Exemplo Página sem Organização

2. Pressione CRTL+U para visualizar o código fonte da página
3. Observe o HTML da página e analise os problemas de organização e indentação

📝 Perguntas para reflexão:

  • ☐ O código está bem indentado?
  • ☐ As tags estão escritas e fechadas corretamente?
  • ☐ Existem comentários úteis no código?
  • ☐ Está fácil entender o que cada parte do código faz?

➡️ Registre suas observações no google forms: https://forms.gle/Yrk81ZX8KYnA53Ev9 Parte 1 — Observações da Inspeção

🛠️ Parte 2 — Corrigindo Código Desorganizado

1. Faça o download do arquivo com o código HTML desorganizado: Baixar código desorganizado (index.html)

2. Acesse seu repositório no GitHub chamado laboratorio-front-end
3. Crie uma nova pasta com o nome: atividade-07-indentacao
4. Dentro dessa pasta, crie um novo arquivo chamado index.html
5. Copie o conteúdo do arquivo baixado, corrija a indentação e organização do HTML
6. Publique normalmente no GitHub Pages

➡️ Envie o link da sua correção funcionando no google forms: Parte 2 - Link do github pages da página corrigida funcionando

📌 Lembre-se:

  • ✅ Corrigir a indentação (usar TAB ou 2 espaços de forma consistente)
  • ✅ Fechar corretamente todas as tags HTML
  • ✅ Adicionar comentários para indicar as seções principais
