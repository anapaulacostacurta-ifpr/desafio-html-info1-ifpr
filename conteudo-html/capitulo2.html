Capítulo 2 – Codificação e Declaração DOCTYPE em Arquivos HTML

Ao criarmos páginas HTML, é fundamental garantir que o conteúdo seja corretamente interpretado e exibido pelos navegadores. Dois conceitos essenciais nesse processo são: a codificação de caracteres e a declaração do tipo de documento (DOCTYPE). Vamos explorar cada um deles!

🧾 O que é codificação em arquivos HTML?

Codificação é o processo que permite que letras, números e símbolos sejam representados por valores numéricos que os computadores conseguem entender.

No contexto de arquivos HTML (e também CSS, JavaScript, etc.), a codificação garante que os textos apareçam corretamente, sem erros ou símbolos estranhos.

🔤 ASCII e Unicode: qual a diferença?

🔹 ASCII (American Standard Code for Information Interchange)

  • É um dos primeiros padrões de codificação.
  • Mapeia um conjunto limitado de caracteres (apenas o alfabeto latino, números e símbolos básicos) a números de 0 a 127.
  • Ainda é utilizado, mas é restrito a conteúdos em inglês ou com poucos caracteres especiais.

🔹 Unicode

  • Criado para suportar uma ampla variedade de idiomas e símbolos.
  • Inclui alfabetos de diferentes línguas (como árabe, chinês, cirílico, etc.), emojis e símbolos técnicos.
  • UTF-8 é a forma mais comum de codificação Unicode na web.

🌍 Por que usar UTF-8 em HTML?

UTF-8 permite que desenvolvedores escrevam páginas em múltiplos idiomas e com diversos caracteres especiais, sem se preocupar com erros de exibição.

É leve, eficiente e compatível com a maioria dos navegadores e sistemas.

Para definir a codificação UTF-8 em um arquivo HTML, usamos a seguinte linha dentro da tag <head>:

<meta charset="UTF-8">

⚠️ Importante: Se a codificação declarada não corresponder à codificação real do arquivo, o navegador pode exibir caracteres incorretamente, como símbolos estranhos ou "caixinhas".

📌 O que é o DOCTYPE em HTML?

A declaração <!DOCTYPE> informa ao navegador qual versão do HTML está sendo utilizada no documento. Essa linha deve sempre aparecer no topo do arquivo HTML, antes de qualquer outra marcação.

📄 Exemplo:

<!DOCTYPE html>

No HTML5, essa é a forma correta e mais simples de declarar o DOCTYPE. Ela garante que o navegador interprete o código de acordo com os padrões modernos.

🏛️ Versões do HTML: por que o DOCTYPE importa?

Versões anteriores do HTML, como HTML 4.01 ou XHTML, exigiam declarações mais longas e específicas.

O HTML5, lançado oficialmente em 2014, trouxe uma sintaxe mais simples e funcionalidades poderosas para construção de sites modernos, acessíveis e responsivos.

Conceito Explicação

ASCII Tabela antiga com suporte limitado a caracteres latinos

UTF-8 Forma mais comum e flexível de Unicode, ideal para web
<meta charset> Tag que define a codificação usada no HTML (geralmente UTF-8)
<!DOCTYPE> Declaração obrigatória no topo do HTML que define a versão do documento
HTML5 Versão atual e recomendada do HTML, moderna e compatível com os padrões
