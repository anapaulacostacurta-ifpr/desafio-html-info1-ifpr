1.4 Organização de Arquivos CSS

1.4.3 Um Único Arquivo CSS Global

Vantagens

  • Centralização – todos os estilos ficam em um único arquivo, favorecendo consistência e reduzindo duplicação.
  • Carregamento mais rápido – o navegador pode cachear um só arquivo, melhorando o tempo de carregamento.
  • Simplicidade – para projetos pequenos, gerenciar um arquivo é mais simples.

Cuidados

  • Organização – à medida que o projeto cresce, manter seções e componentes separados dentro do mesmo arquivo pode ficar difícil.
  • Conflitos – uma alteração global pode afetar páginas/partes não previstas.

1.4.4 Abordagem Híbrida

Mantenha estilos comuns em um CSS global e, para áreas complexas ou páginas específicas, crie folhas de estilo separadas. Assim, você equilibra reuso (global) e modularidade (por seção).

Regra prática: projetos grandes tendem a se beneficiar de CSS por seção/ componente; projetos menores ganham com um arquivo único para consistência.

Sempre avalie escalabilidade, legibilidade e colaboração ao decidir.

1.4.5 Exemplo de Estrutura de Catálogo

Uma estrutura simples e modular para HTML/CSS:

project-folder/
├─ index.html                ← página inicial
├─ about/                    ← seção “Sobre”
│  ├─ about.html
│  └─ about.css
├─ services/                 ← seção “Serviços”
│  ├─ services.html
│  └─ services.css
├─ contact/                  ← seção “Contato”
│  ├─ contact.html
│  └─ contact.css
├─ css/
│  └─ styles.css             ← folha principal (global)
├─ js/
│  └─ script.js              ← scripts
├─ images/
│  ├─ logo.png
│  └─ banner.jpg
└─ fonts/
   ├─ font1.ttf
   └─ font2.woff

Essa estrutura separa arquivos por tipo/função e organiza o site em diretórios por seção, facilitando manutenção e evolução.

1.6 Comentários em CSS

1.6.1 Introdução

Comentários documentam o CSS, explicam decisões e orientam quem lê o código. Eles são ignorados pelo navegador, mas visíveis no código-fonte.

1.6.2 Como Criar Comentários

/* Este é um comentário de uma linha */
selector {
  property: value; /* Comentário ao lado de uma declaração */
}

/*
  Comentário de múltiplas linhas:
  útil para blocos maiores de explicação.
*/

1.6.3 Por que Usar Comentários

  • Documentar decisões (ex.: “aumentar fonte do <h1> por acessibilidade”).
  • Desativar trechos temporariamente durante testes.
/* Desativando regras temporariamente:
h1 {
  font-size: 24px;
  color: red;
}
*/

Boas práticas: explique o essencial, evite óbvio, atualize comentários ao alterar código e mantenha um tom profissional.

1.7 Seletores: Alvejando Elementos HTML

1.7.1 Tipos de Seletores

Seletores definem o que será estilizado. Conhecer seus tipos é fundamental.

1.7.2 Seletor de Elemento (tipo)

Alveja todas as ocorrências de uma tag.

/* todos os parágrafos ficam vermelhos */
p { color: red; }

/* múltiplos elementos com as mesmas regras */
h1, h2, p { color: #333; }

1.7.3 Seletor de Classe

Aplica estilos a elementos com o atributo class. Usa-se com “.” (ponto).

/* CSS */
.button {
  background-color: blue;
  color: white;
}

/* HTML */
<button class="button">Enviar</button>
<a class="button" href="#">Link estilizado</a>

1.7.4 Seletor de ID

Aponta um único elemento com id único por página. Usa-se com “#”.

/* CSS */
#header {
  font-size: 24px;
  color: green;
}

/* HTML */
<div id="header">Cabeçalho</div>

Dica: use classes para estilização recorrente e IDs para ancoragens, acessibilidade ou ganchos específicos. IDs têm maior especificidade que classes.

1.7.5 Seletores de Atributo

Estilizam com base na presença/valor de atributos.

/* qualquer elemento com data-highlight */
[data-highlight] { background: yellow; }

/* correspondência exata */
[data-highlight="true"] { background: yellow; }

/* contém substring */
[class*="button"] { border: 1px solid #000; }

/* prefixo */
[href^="https://"] { color: blue; }

/* sufixo */
[src$=".png"] { border: 1px solid #000; }

1.7.6 Pseudo-classes, Pseudo-elementos e Combinadores

  • Pseudo-classes (ex.: :hover, :focus) – estados dinâmicos.
  • Pseudo-elementos (ex.: ::before, ::after) – partes do conteúdo.
  • Combinadores: espaco (descendentes), > (filhos diretos), + (irmão adjacente).

Veja mais em MDN Web Docs quando quiser se aprofundar.

1.8 Cascade, Specificity e Herança

1.8.1 Visão Geral

Esses princípios determinam como regras entram em vigor quando há conflitos.

1.8.2 A Cascade do CSS

Quando regras conflitam, o navegador considera a origem (autor/usuário/UA), a ordem de declaração e a especificidade. De forma geral: inline > ID > classe/atributo/pseudo-classe > elemento.

1.8.3 Especificidade

Quanto mais específico o seletor, maior a prioridade. Ex.: #header vence .header se houver conflito.

1.8.4 Herança

Algumas propriedades (tipografia e cor) são herdadas pelos filhos; layout normalmente não.

/* CSS */
.parent {
  font-family: Arial, sans-serif;
  font-size: 16px;
  border: 1px solid black;
  padding: 10px;
}

/* HTML */
<div class="parent">
  <p>Este parágrafo herda fonte e tamanho.</p>
</div>

1.8.5 Sobrescrevendo Propriedades Herdadas

/* herdado da .parent: font-family e (inicialmente) font-size */
.parent { font-family: Arial, sans-serif; font-size: 16px; }

/* o p abaixo sobrescreve o font-size e a cor */
p {
  font-size: 20px;
  color: green;   /* (correto) – "font-color" não existe em CSS */
}

1.8.6 Controlando Estilo com !important

.heading { color: blue !important; }

Use com moderação. Pode ser útil para sobrepor estilos de UA ou bibliotecas de terceiros, mas o excesso dificulta manutenção. Prefira estrutura e especificidade bem planejadas; se usar, documente o motivo.
